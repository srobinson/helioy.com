Helioy Context Manifesto

LLMs do not primarily suffer from a lack of intelligence. They suffer from a lack of usable context.

Raw environments are too large, too noisy, too expensive, and too undifferentiated to be the primary substrate for finite-context intelligence.

  • Codebases are too big to read linearly.
  • Documentation is too verbose to paste wholesale.
  • Conversation history is too messy to trust as future memory.

Product Thesis

Helioy exists to turn raw environments into high-signal context for finite-context intelligence. Not search for its own sake. Not memory for its own sake. Not orchestration for its own sake. Context. Useful, compressed, structured, trustworthy context.

The Product Group

These are not separate ideas. They are different answers to the same problem.

fmm

Solves code context. Compiles code into structural metadata so models don't rediscover structure from scratch.

What does the codebase structurally contain?

mdcontext

Solves document context. Compiles documents into indexed sections, ranked results, and token-bounded context.

What do docs mean within a finite budget?

AM

Solves reflective memory. Forms memory through conversation to track what becomes true through repeated dialogue.

What is becoming true over time?

nancyr

The orchestration harness. Decides how context moves, what passes to which agent, and coordinates action.

How does context become coordinated action?

The Three Tenets

01

Context is Finite

Tokens are a strict budget. Every token spent on boilerplate or irrelevant context is a token stolen from reasoning depth. Compression is not optional.

02

Structure Beats Search

Raw text search is lazy. Intelligence requires graphs, hierarchies, and typed references. If a model has to deduce the structure of a codebase, the context engine has failed.

03

Memory Must Compound

A conversation history is a transcript, not a memory. True memory is distilled, synthesized, and structurally updated to make future interactions more efficient.

fmm: Frontmatter Matters

Structural intelligence for codebases. The main value is faster orientation. fmm maintains a single SQLite database at the project root indexing exports, imports, dependencies, line counts, and file outlines.

Without fmm

Orient with grep, listings, and selective file reads

Structure is reconstructed from scattered results

With fmm

Query structure first, then open needed files

See directory shape, key files, and dependency impact up front

mdcontext

Documentation is traditionally consumed linearly. For finite-context intelligence, it must be consumed hierarchically and budget-consciously. mdcontext is an AST-aware semantic chunker and token-bounded retrieval engine.

AST-Aware Chunking

Breaks markdown at logical header boundaries, preserving parent-child semantic relationships rather than arbitrary character limits.

Token Bounding

Retrieval strictly respects context window budgets. If a requested section is too large, it automatically summarizes or intelligently truncates.

Semantic Routing

Maps user queries to the most relevant document nodes using fast, local embeddings before any full LLM synthesis happens.

Nancy V2

A self-improving runtime for collaborative intelligence. It coordinates specialized agents, routes context, preserves coherence under pressure, and improves teamwork over time.

Where This Started

Before nancyr, there was nancy: an autonomous task execution loop with context awareness and token management. The current runtime grows out of that earlier experiment in iterative agent work.

The Optimization Target

The deeper optimization target is quality of collaboration. `nancyr` should optimize how intelligence is organized, not just how prompts are written.

  • Division of labor
  • Timing of intervention
  • Precision of handoffs
  • Correctness of escalation

The Four Learning Loops

1. Task Loop

Did the team complete the task well?

Signals: correctness, speed, rework, regressions

2. Coordination Loop

Was the work split correctly?

Signals: duplicate effort, poor sequencing, blocked deps

3. Protocol Loop

Did the protocol help or hinder?

Signals: bus message quality, escalation policy clarity

4. Identity Loop

Are the roles themselves right?

Signals: recurring confusion, missing specialist roles

Final Thesis

`nancyr` should become a self-improving runtime for collaborative intelligence, where specialized agents, structured context, reflective memory, and explicit coordination protocols combine into compounding execution quality over time.
